Nginx

Nginx 在启动后，包含一个 master 进程和多个 worker 进程。

master 进程主要用来管理 worker 进程，包含：接收来自外界的信号，向各 worker 进程发送信号，监控 worker 进程的运行状态，当 worker 进程退出后(异常情况下)，会自动重新启动新的 worker 进程。

而网络请求则是由 worker 进程来处理。所有的worker 进程之间是平等的，并各进程互相独立。一个请求，只可能在一个 worker 进程中处理，一个 worker 进程，不可能处理其它进程的请求。worker 进程的个数会设置为cpu核心数，目的是充分利用 CPU 资源，同时，避免进程数量过多，避免进程竞争 CPU 资源，增加上下文切换的损耗。

Nginx启动运行过程

  1. master建立listen的socket（listenfd）
  2. master进程fork出多个worker进程（根据设定的worker进程数）
  3. 新请求到来，所有worker的listenfd权限变为可读
  4. 所有worker进程竞争accept_mutex，拿到锁的woker进程注册listenfd的读事件
  5. worker进程accept当前连接并处理请求

Nginx特点

基于事件

请求进来之后，一个worker进程接手处理，当无法立即返回结果时（两种可能：IO阻塞，转发后端处理），就直接返回注册一个事件，然后接着去处理其他的请求，一旦结果返回，就会触发这个事件，worker继续接手。

异步非阻塞

建立连接之后需要读取数据和发送数据，具体到系统底层就是一个IO事件，如果在IO准备好之前，线程挂起等待，直到IO准备好之后才继续执行就称为阻塞。与之相反，非阻塞是如果IO还没有准备好，就直接返回，先去处理其他任务。

同步是当函数调用发出后，一直等待调用的结果。异步是当函数调用发出后没有得到返回结果，以通知的方式获得结果。

如果事件没有准备好，就立即返回，线程就可以去处理其他请求，等事件准备好之后，再去接着执行，这就是异步非阻塞。

动静分离

为什么动静分离：Nginx的静态处理能力很强，但是动态处理能力不足。

动静分离原理：对于静态资源，在反向代理服务器nginx中进行缓存。若用户请求的动态文件，则转发给Tomcat服务器处理。

Nginx配置

Nginx配置文件结构

main（全局设置）：设置的指令将影响其他所有设置；
server（主机设置）：指令主要用于指定主机和端口
upstream（负载均衡服务器设置）：指令主要用于负载均衡，设置一系列的后端服务器
location（URL匹配特定位置的设置）：用于匹配网页位置。

server继承main，location继承server，upstream既不会继承其他设置也不会被继承。

全局块

全局块的配置影响Nginx全局。

全局块包含的配置有：

user：指定运行Nginx的用户和组

worker process：指定worker进程数（建议和CPU的数量一致）

error_log：错误日志位置

pid：pid文件存放路径

worker_rlimit_nofile：指定进程可以打开的最多文件描述数目

events块

events块包含的配置有：

use：指定Nginx的工作模式。Nginx支持的工作模式有select、poll、kqueue、epoll、rtsig和/dev/poll 。其中select 和poll 都是标准的工作模式，kqueue和epoll是高效的工作模式，不同的是epoll用在Linux平台上，而kqueue用在BSD系统中。对于Linux系统，epoll工作模式是首选。

worker_connections：定义Nginx每个进程的最大连接数，默认是1024。

http全局块

http块包含的配置有：

include：引入其他Nginx配置或第三方模块。

log_format：Nginx日志配置。

sendfile：是否开启高效文件传输模式。

keepalive_timeout ：连接超时时间。

server块

server 块是对虚拟主机的配置。

listen：监听端口。

server_name：服务器的访问地址，ip或域名。

index：默认网页。

root：虚拟主机的网页根目录。

charset：网页的默认编码格式

access_log：虚拟主机的访问日志存放路径和输出格式

location块

针对某个URL的控制块，过滤URL可以使用正则表达式。

root：网页根目录。

index：默认网页。

proxy_pass：对该块对应的URL进行反向代理，设置代理的URI，将请求转发到对应的服务器（ip或者域名），也可以是upstream组名，将请求进行负载均衡

proxy_set_header：修改发往后端服务器的请求头。（X-Real-IP和X-Forwarded-For配合使用将客户端真实IP告知后端服务器）

expires：过期时间，过期会自动清除

upstream块

配置后台的真实服务器的负载均衡。

模式：默认轮询，可指定ip_hash模式或者least_conn模式。

server：后台真实服务器的地址，多个，可以设置服务器权重weight。

Nginx既可以配置四层负载均衡也可以配置七层负载均衡，四层负载均衡是根据ip和端口进行转发，而七层负载均衡根据应用层的具体内容进行转发，http协议就是根据http header的内容进行转发，比如URL、浏览器类型、语言等。

反向代理及配置

反向代理是以代理服务器来接受互联网上的连接请求，然后将请求转发给内部网络上的服务器。Nginx 的优势是在于它的异步非阻塞模型，可以通过基于事件的方式同时处理和维护多个请求，而后端只要去做逻辑计算，节约等待时间去处理更多请求。nginx反向代理的基本配置可以在nginx.conf中配置一个location块，利用proxy_pass配置代理的服务器地址（IP或域名，指代后端服务器），可以利用proxy_set_header 指令来修改http头。

负载均衡策略

Nginx内置的负载均衡策略有轮询，加权轮询，哈希分配（Ip_hash），最少连接数（last_conn），URL哈希。

轮询就是将请求依次平均的分发到后端服务器。

加权轮询给每台后端服务器设置不同的权重，根据权重给不同服务器分发不同数量请求。

哈希分配是对客户端请求的ip进行hash操作，然后根据hash结果将同一个客户端ip的请求分发给同一台服务器进行处理，可以解决session不共享的问题。

最少连接数是将请求分发给连接数最少的后端服务器。

Nginx的负载均衡需要配置upstream块和反向代理。
