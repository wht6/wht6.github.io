Docker基础命令

选项

-p xxx:yyy ：将容器的yyy端口指定映射到宿主机的xxx端口上，-p yyy将容器的yyy端口映射到宿主机的任意端口。

-v <目录1>:<目录2>：将宿主机目录1挂载到容器目录2。

-d：后台运行。

-it：交互式启动。

-network=xxx：指定网络模式。bridge，默认模式，将容器网络和宿主机网络通过网桥连接。host，容器使用宿主机的网络。container:NAME_or_ID，使用其他容器的网络。none，不配置网络，需要在容器中自定义网络。

—name：指定容器名。

—gpus=all：支持显卡。

操作

docker pull：下载镜像

docker push：将本地镜像上传到镜像仓库

docker info：展示docker相关的信息

docker inspect：获取容器或镜像的元数据

docker images -a：列出所有镜像

docker rmi -f：删除镜像

docker rm -f：删除容器

docker create：创建容器但是不启动

docker run：创建容器并运行一个命令

docker ps -a：列出所有容器

docker start/restart：启动/重启容器

docker stop：停止容器

docker kill：强制停止容器

docker exec：进入容器。

docker cp <文件1>:<目录2>：将宿主机文件1拷贝到容器目录2

docker logs：查看容器标准日志输出

docker build：通过Dockerfile构建镜像

上面docker build命令中最后的“.”表示构建镜像上下文的路径，docker build 命令得知这个路径后，会将路径下的所有内容打包，然后上传给 Docker 引擎（在docker服务端）。这样 Docker 引擎收到这个上下文包后，展开就会获得构建镜像所需的一切文件。所以COPY指令也是复制上下文中文件，因此COPY的源文件全是相对路径。

这行命令指定了构建所需的 Git repo，并且指定默认的 master 分支，构建目录为 /8.14/，然后 Docker 就会自己去 git clone 这个项目、切换到指定分支、并进入到指定目录后开始构建。

如果所给出的 URL 不是个 Git repo，而是个 tar 压缩包，那么 Docker 引擎会下载这个包，并自动解压缩，以其作为上下文，开始构建。

docker login：登录到镜像仓库

docker logout：退出登录的镜像仓库

docker save -o：将镜像以文件形式保存到本地

docker load：从本地文件中加载镜像

Dockerfile命令

FROM 表示构建镜像的基础镜像

Docker 存在一个特殊的镜像scratch。这个镜像是虚拟的概念，并不实际存在，它表示一个空白的镜像。

如果你以 scratch 为基础镜像的话，意味着你不以任何镜像为基础，接下来所写的指令将作为镜像第一层开始存在。

不以任何系统为基础，直接将可执行文件复制进镜像的做法并不罕见，比如swarm、coreos/etcd。对于 Linux 下静态编译的程序来说，并不需要有操作系统提供运行时支持，所需的一切库都已经在可执行文件里了，因此直接 FROM scratch 会让镜像体积更加小巧。使用 Go 语言开发的应用很多会使用这种方式来制作镜像，这也是为什么有人认为 Go 是特别适合容器微服务架构的语言的原因之一。

MAINTAINER 表示此docker镜像维护者的信息

RUN 用于执行命令操作，通常用来配置镜像所需的环境

Dockerfile 中每一个指令都会建立一层，RUN 也不例外。因此，为了减少中间镜像层数，通常将多个命令合并到一起执行，&&连接，并可以用\换行。并且最好在最后清理所有下载的文件，apt缓存文件等无关的东西，减小构建镜像大小。

WORKDIR 用于切换工作路径，即当前路径。

CMD 用于指定默认的容器主进程的启动命令。有三种执行模式

【CMD 实在容器创建即docker run阶段的时候执行的，RUN命令是在Dockerfile build的阶段执行的】

ENV 指定用来设置环境变量，类似于在系统定义变量，在后续的系统的可以使用声明的环境变量

ADD指令用来从上下文目录中复制文件到容器的指定路径

COPY 指令，从上下文目录中复制文件或者目录到容器里指定路径

<目标路径> 可以是容器内的绝对路径，也可以是相对于工作目录的相对路径（工作目录可以用 WORKDIR 指令来指定）。目标路径不需要事先创建，如果目录不存在会在复制文件前先行创建缺失目录。

使用 COPY 指令，源文件的各种元数据都会保留。比如读、写、执行权限、文件变更时间等。

ADD与COPY 的区别

  1. *ADD与COPY 指令都可以将文件资源移动到容器之中，
  2. ADD 的源地址可以是url，会自动下载到目标路径，COPY只能复制宿主机的文件
  3. ADD 指令如果拷贝的资源是一个可识别的压缩格式的话，会自动解压文件

ADD像是一个加强的版的COPY指令，但是实际使用中比较少。

ENTRYPOINT 用于指定容器运行程序和参数

RUN、CMD和ENTRYPOINT比较：

  • RUN命令执行命令并创建新的镜像层，通常用于安装软件包
  • CMD命令设置容器启动后默认执行的命令及其参数，但CMD设置的命令能够被docker run命令后面的命令行参数替换
  • ENTRYPOINT配置容器启动时的执行命令（不会被忽略，一定会被执行，即使运行 docker run时指定了其他命令）

VOLUME 用于挂载数据卷，即挂载宿主机目录。属于匿名挂载，容器自动指定宿主机的目录。

ARG 定义创建镜像过程中使用的变量。镜像创建成功后变量将不再存在。

AS 多阶段构建镜像的时候，为阶段命名
